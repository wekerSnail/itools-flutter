# 系统托盘文档（统一版）

本文件整合了此前多个托盘相关文档（诊断、修复、快速参考、测试、最终排障），作为唯一维护入口。

## 1. 当前实现概览（基于代码）

托盘核心实现位于 `lib/core/system/app_tray_service.dart`，当前机制包括：

- Windows 平台初始化与监听注册（`TrayListener`、`WindowListener`）
- 窗口关闭时隐藏到托盘（非直接退出）
- 托盘菜单：打开主界面、备份还原、开机自启、退出
- 多路径图标探测（开发态与发布态）
- 每 30 秒托盘恢复检查（降低长时运行丢失概率）
- 统一日志输出，便于定位初始化与交互问题

入口调用在 `lib/main.dart` 的窗口显示后阶段。

## 2. 常见症状与优先排查路径

### 症状 A：托盘不显示

优先按以下顺序检查：

1. **Windows 隐藏托盘区域**（右下角上箭头）
2. **通知区域图标设置**（是否被设为隐藏）
3. **图标文件是否存在**：`assets/tray_icon.ico`
4. **应用日志**（`log_new.txt`）中是否有关键成功日志
5. 以管理员身份运行（排除权限限制）

### 症状 B：运行久后托盘消失或无响应

1. 检查日志中是否持续出现托盘恢复相关输出
2. 验证右键菜单与左键打开窗口是否仍可触发
3. 若偶发失败，优先收集日志后再调整恢复策略（例如检查周期）

## 3. 关键日志关键词

- `[Tray] Starting initialization...`
- `[Tray._setupTray] ✓ Found icon:`
- `[Tray._setupTray] ✓ Icon set successfully`
- `[Tray._setupTray] ✓ Setup complete!`
- `[Main] ✓ Tray service initialized successfully`

若出现 `✗` 或 `failed`，按对应错误上下文定位资源路径、权限或系统托盘行为。

## 4. 测试清单（开发与发布通用）

- [ ] 启动后托盘图标可见
- [ ] 左键托盘可唤起主窗口
- [ ] 右键托盘可弹出菜单
- [ ] 菜单“开机自启”切换正常
- [ ] 菜单“退出”可正常退出应用
- [ ] 后台运行 30 分钟后托盘仍可用
- [ ] 发布版（`build/windows/x64/runner/Release`）行为一致

## 5. 速查：快速恢复动作

1. 结束 `itools.exe` 进程后重启应用
2. 刷新 Windows 资源管理器（必要时）
3. 检查并恢复通知区域图标显示策略

## 6. 变更维护建议

- 若调整托盘逻辑，请同步更新本文件“当前实现概览”与“测试清单”。
- 不再新增 `TRAY_*.md` 平行文档，避免知识分叉。
