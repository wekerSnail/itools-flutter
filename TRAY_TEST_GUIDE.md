## 🚀 快速开始

### 1. 构建测试版本

```bash
# 清理之前的构建
flutter clean

# 重新构建
flutter build windows

# 或者直接运行开发版本
flutter run -d windows
```

### 2. 验证托盘初始化

运行应用后，查看控制台输出：

✅ **预期看到**：

```
[Tray] icon set to: C:\...\assets\tray_icon.ico
[Tray] setup complete
```

❌ **如果看到错误**：

```
[Tray] icon file not found at: ...
```

- 检查 `assets/tray_icon.ico` 文件是否存在
- 文件应该是32x32像素的ICO格式

### 3. 测试功能

| 操作                 | 预期结果                   | 通过 |
| -------------------- | -------------------------- | ---- |
| **左键点击托盘**     | 打开主界面                 | ✓    |
| **右键点击托盘**     | 弹出菜单                   | ✓    |
| **点击"打开主界面"** | 显示窗口                   | ✓    |
| **点击"开机自启"**   | 菜单文字改为"取消开机自启" | ✓    |
| **点击"退出"**       | 应用正常退出               | ✓    |

### 4. 验证恢复机制

**测试长期运行稳定性**：

```
1. 运行应用
2. 最小化应用到系统托盘
3. 等待 2 分钟（足够触发4次恢复检查）
4. 检查托盘图标是否仍存在
5. 尝试点击托盘（应该能打开窗口）
```

**检查日志**（应该每30秒出现一次）：

```
[Tray] setup complete
```

### 5. 压力测试

运行以下场景测试应用稳定性：

```
① 启动 -> 最小化 -> 等待 -> 打开 -> 关闭 -> 重复5次
② 启动 -> 托盘右键菜单点击 -> 各菜单项测试
③ 长期后台运行（建议 8+ 小时）-> 时间检查托盘
④ 应用挂起恢复 -> 托盘是否还存在
```

## 🐛 故障排查

### 问题1: 托盘仍然不显示

**检查清单**：

- [ ] 是否看到 `[Tray] setup complete` 日志？
- [ ] 如果没有，是否看到 `[Tray] icon file not found` 错误？
- [ ] `assets/tray_icon.ico` 文件是否存在？
- [ ] 文件是否是有效的ICO格式？

**解决方案**：

```bash
# 检查文件存在
ls -la assets/tray_icon.ico

# 确保assets文件夹在pubspec.yaml中声明
flutter pub get

# 清理重建
flutter clean
flutter pub get
flutter run -d windows
```

### 问题2: 托盘偶尔消失

**检查清单**：

- [ ] 是否看到 `[Tray] recovery check failed` 错误？
- [ ] 应用是否处于睡眠/挂起状态？
- [ ] 系统资源是否充足？

**解决方案**：

```dart
// 如果恢复频率不够，可修改间隔
// lib/core/system/app_tray_service.dart
const Duration(seconds: 15),  // 改为15秒（更频繁）
```

### 问题3: 内存持续增长

**检查清单**：

- [ ] Timer是否正确取消？
- [ ] dispose()方法是否被调用？

**解决方案**：
确保应用退出时调用dispose()

```dart
// 在app.dart或main.dart的清理代码中
await AppTrayService.instance.dispose();
```
